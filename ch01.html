En el último par de años el desarrollo web ha visto un enorme advenimiento de la Single Page Application ( SPA ). Al principio era simple-recargar una página completa para llevar a cabo un cambio en la pantalla o realizar una acción del usuario. El problema con esto era el gran tiempo de ida y vuelta de la solicitud completa para llegar al servidor web y la vuelta al cliente.

Web development has seen a huge advent of Single Page Application (SPA) in the past couple of years. Early development was simple—reload a complete page to perform a change in the display or perform a user action. The problem with this was a huge round-trip time for the complete request to reach the web server and back to the client.

Luego vino AJAX, que envió una solicitud al servidor, y podía actualizar partes de la página sin volver a cargar la página actual. Se mueve en la misma dirección, vimos la aparición de las SPAs.

Then came AJAX, which sent a request to the server, and could update parts of the page without reloading the current page. Moving in the same direction, we saw the emergence of the SPAs.

Encapsulando el contenido frontend pesado y entregandolo al navegador del cliente una sola vez, mientras se mantiene un pequeño canal para la comunicación con el servidor basado en cualquier evento; esto se suele completar con una API delgada en el servidor web.

Wrapping up the heavy frontend content and delivering it to the client browser just once, while maintaining a small channel for communication with the server based on any event; this is usually complemented by thin API on the web server.

El crecimiento en tales aplicaciones se ha complementado con las bibliotecas y los frameworks como Ext JS, KnockoutJS, BackboneJS, AngularJS, EmberJS JavaScript, y más recientemente, React y Polymer.

What is React?

ReactJS trata de resolver el problema de la capa Vista. Puede muy bien ser definido y usado como la V en cualquiera de los frameworks MVC. No es pertinaz sobre la forma en que se debe utilizar. Se crean representaciones abstractas de vistas. Se descomponen partes de la vista de los componentes. Estos componentes abarcan tanto la lógica para manejar la pantalla de vista y la vista en sí. Podes contener datos que se utilizan para representar el estado de la aplicación.

ReactJS tries to solve the problem from the View layer. It can very well be defined and used as the V in any of the MVC frameworks. It's not opinionated about how it should be used. It creates abstract representations of views. It breaks down parts of the view in the Components. These components encompass both the logic to handle the display of view and the view itself. It can contain data that it uses to render the state of the app.

Para evitar la complejidad de las interacciones y el posterior renderizado del procesamiento requerido, React realiza un procesamiento completo de la aplicación. Mantiene un simple flujo de trabajo.

To avoid complexity of interactions and subsequent render processing required, React does a full render of the application. It maintains a simple flow of work.

React se funda en la idea de que la manipulación del DOM es una operación costosa y debe ser minimizada. También reconoce la optimización de la manipulación del DOM por parte dará lugar a un montón de código boilerplate, que es propenso a errores, aburrido, y repetitivo.

React is founded on the idea that DOM manipulation is an expensive operation and should be minimized. It also recognizes that optimizing DOM manipulation by hand will result in a lot of boilerplate code, which is error-prone, boring, and repetitive.

React resuelve esto dandole al desarrollador un DOM virtual para renderizar en lugar del DOM real. Si encuentra diferencia entre el DOM real y el DOM virtual y lleva a cabo el número mínimo de operaciones DOM necesarias para alcanzar el nuevo estado.

React solves this by giving the developer a virtual DOM to render to instead of the actual DOM. It finds difference between the real DOM and virtual DOM and conducts the minimum number of DOM operations required to achieve the new state.

React también es declarativo. Cuando se cambia de datos, React golpea conceptualmente el botón de actualización para actualizar y conoce solamente las partes que han cambiado.

React is also declarative. When the data changes, React conceptually hits the refresh button and knows to only update the changed parts.

Este simple flujo de datos, unido a una lógica de visualización simple y muerta, hace que el desarrollo con ReactJS sea sencillo y facil de entender.

This simple flow of data, coupled with dead simple display logic, makes development with ReactJS straightforward and simple to understand.
