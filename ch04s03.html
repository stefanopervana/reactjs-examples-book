Getting started with forms

"Formas comportan ligeramente diferente en el mundo React que en el mundo HTML normal. Son un poco más especial que los otros componentes del DOM en el mundo React. El <input>, <textarea> y <option> las etiquetas son algunos de los componentes de entrada comunes proporcionados por React." explica Mike.

"Forms behave slightly differently in the React world than in the normal HTML world. They are a bit special than the other DOM components in the React world. The <input>, <textarea>, and <option> tags are some of the common input components provided by React." explained Mike.

"Estos componentes forman están mutados cuando el usuario interactúa con ellos, añade un poco de texto en la entrada, o selecciona una opción. Por lo tanto, tenemos que tener cuidado de que estamos manejando esas interacciones del usuario correctamente." Mike explica con más detalle.

"These form components are mutated when the user interacts with them, adds some text in the input, or selects an option. Therefore, we need to take care that we are managing those user interactions properly." Mike explained further.

"Vamos a empezar con una simple etiqueta de entrada para comprender la interacción del usuario." informó Mike.

"Let's start with a simple input tag to understand the user interaction." informed Mike.

// src/index.js

import React from 'react';
import ReactDOM from 'react-dom';

var InputExample = React.createClass({
  render() {
    return (
      <input type="text" value="Shawn" />
    );
  }
});
ReactDOM.render(<InputExample />,  
                 document.getElementById('root'));

"También estamos utilizando una nueva manera de definir funciones en ES6."

"We are also using a new way of defining functions in ES6."

// Old way of defining function
render: function() {
  return (
    <input type="text" value="Shwn" />
  );
}

// New way of defining function
render() {
    return (
      <input type="text" value="Shawn" />
    );
  }

"El uso de esta nueva sintaxis, que no tenemos que escribir la palabra function cada vez que se define la función."

"Using this new syntax, we don't have to write word function every time we define the function."
