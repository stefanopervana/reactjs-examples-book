"Shawn, React utiliza DOM virtuales para realizar un seguimiento de lo que ha cambiado en el DOM real. Es concepto es muy fácil de entender. React siempre mantiene una copia de la representación del DOM real en la memoria. Cada vez que cambia algo, como algún estado la manipulación, se calcula otra copia del DOM que se generará con el nuevo estado y los accesorios. Luego se calcula la diferencia entre la copia original del DOM virtual y la nueva copia del DOM virtual. Esta diferencia de resultados en las operaciones de mínimos sobre el verdadero DOM que podes tomar el DOM actual a una nueva etapa. de esta manera, React no tiene que hacer grandes cambios cuando algo cambia." "Shawn, React uses virtual DOM to keep track of what has changed in the real DOM. It's concept is very easy to understand. React always keeps a copy of the representation of the actual DOM in memory. Whenever something changes, such as some state manipulation, it calculates another copy of the DOM that will be generated with new state and props. Then it calculates the difference between the original copy of the virtual DOM and the new copy of the virtual DOM. This difference results in minimal operations on the real DOM that can take the current DOM to a new stage. In this way, React does not have to do major changes when something changes." Mike explained.
