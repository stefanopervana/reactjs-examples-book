"Shawn, vamos a hablar de una técnica más de manejo de datos en un componente, el estado. Cada componente puede tener su propio estado en React. La principal diferencia entre el estado y los props es que los props se pasan al componente desde el componente principal; mientras que el estado es algo que es interno al componente.

"Shawn, let's talk about one more technique of handling data in a component, state. Every component can have its own state in React. The main difference between state and props is that props are passed to the component from the parent component; whereas, state is something that is internal to the component.

Los props se pasan cuando en un componente se crea una instancia. El estado es algo que podes cambiar con el tiempo. Por lo tanto, los cambios en el estado afectan a la representación de los componentes. Tené en cuenta al estado como una especie de estructura de datos privada del componente." Añadió Mike.

Props are passed when a component gets instantiated. State is something that can change over time. Therefore, changes in state affect the rendering of components. Consider state as some sort of private data structure of the component." Mike added.
