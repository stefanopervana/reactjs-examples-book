State versus props

"Shawn, es importante entender la diferencia entre los props y el estado y dónde utilizarlo." informó Mike.

"Shawn, it's important to understand the difference between props and state and where to use what." informed Mike.

"Los props son inmutables. Ellos no deben ser actualizados por el componente al que se pasan. Son propiedad del artículo que les pasa a algún otro componente. El estado es algo interno y privado para el componente. El estado puede y va a cambiar dependiendo en las interacciones con el mundo exterior". dijo Mike.

"Props are immutable. They should not be updated by the component to which they are passed. They are are owned by the component which passes them to some other component. State is something internal and private to the component. State can and will change depending on the interactions with the outer world." said Mike.

"El estado debe almacenar datos tan simples como sea posible, como si una casilla de entrada está marcada o no o una clase CSS oculta o muestra el componente". Mike añadió.

"State should store as simple data as possible, such as whether an input checkbox is checked or not or a CSS class that hides or displays the component." Mike added.

"Otra cosa es asegurarse de no repetir los props en el estado." dijo Mike.

"Another thing to make sure is to not duplicate props in state." said Mike.

"Es posible establecer el estado basado en datos pasados ​​en los props. Sin embargo, el componente principal puede actualizar los props y enviarlos de nuevo. En este caso, el estado va a ser confuso, con nuevos datos si ha habido algún cambio en el estado."

"It is possible to set the state based on data passed in props. However, the parent component can update the props and send them again. In this case, the state will be muddled up with new data if there have been any changes to the state."

"Además, los datos están presentes en dos lugares ahora, por lo tanto, se hace más difícil de manejar dos fuentes de datos." Mike explicó.

"Also, the data is present at two places now, therefore, it becomes harder to manage two sources of data." Mike explained.

"Creo que en estos casos, lo mejor es usar accesorios directamente, ¿verdad?" Preguntó a Shawn.

"I think that in such cases, it's best to use props directly, right?" Shawn asked.

"Sí. El estado es totalmente opcional. Lo mejor es evitarlo en la medida de lo posible. En eso tienes razón." Mike dijo feliz.

"Yes. The state is entirely optional. It's best to avoid it as much as possible. You got that right." Mike said happily.
