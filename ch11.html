"Sin embargo, los navegadores actuales todavía no entienden el código ES6 o ES7 que hemos escrito. Utilizamos Babel para transformar este código en JavaScript ES5, que los navegadores actuales podes ejecutar. Se nos permite utilizar la sintaxis de JavaScript del futuro hoy. Babel también tiene soporte JSX, por lo tanto, es muy útil para usar con React ". explica Mike.

"However, current browsers still don't understand the ES6 or ES7 code that we have written. We use Babel to transform this code in ES5 JavaScript, which current browsers can run. It allows us to use the JavaScript syntax from the future today. Babel also has JSX support, therefore, it is very handy to use with React." explained Mike.

"Babel es muy modular y viene con una arquitectura plug-in. Cuenta con plugins para diferentes sintaxis ES6 / ES7. En general, queremos utilizar React plugins específicos específicos y ES6. Grupos Babel tales complementos comunes en algo llamado preajustes . Babel tiene varios plugins para ES6, React, así como las diferentes etapas de las futuras propuestas de lenguaje ".

"Babel is very modular and comes with a plugin architecture. It has plugins for different ES6/ES7 syntaxes. Generally, we want to use React specific and ES6 specific plugins. Babel groups such common plugins into something called presets. Babel has various plugins for ES6, React, as well as different stages of future language proposals."

"Estamos sobre todo interesados en utilizar el ES2015 y React preajustes, que forman parte de todos los plugins relacionados con la ES6 y React. De vez en cuando, necesitamos algunas de las características avanzadas, tales como la sintaxis de la función bind ES7, por lo que necesitamos para configurarlo por separado. En tales casos, se utilizan los plugins individuales directamente, ya que utilizamos transform-function-bind para la función de bind sintaxis ".

"We are mostly interested in using the ES2015 and React presets, which comprise of all the plugins related to ES6 and React. Occasionally, we do need some of the advanced features, such as the ES7 function bind syntax, so we need to configure it separately. In such cases, we use the individual plugins directly, as we used transform-function-bind for the function bind syntax."

Note

Todos estos ajustes preestablecidos y plugins vienen en sus propios paquetes de NGP. Babel se construye de esta manera, un pequeño núcleo y una arquitectura de plugins gigante con una gran cantidad de opciones de configuración a su alrededor.

All of these presets and plugins come in their own npm packages. Babel is built this way—a small core and a giant plugin architecture with a lot of configuration options around it.

"Por lo tanto, tendremos que instalar todos estos paquetes por separado."

"Therefore, we will have to install all of these packages separately."

npm install babel-preset-es2015 --save

"Linting is very important, especially if we want to maintain the code quality across different projects. Fortunately, linting React projects is very easy with ESLint. It also supports ES6 syntax and JSX so that we can also lint our next generation code." informed Mike.

"Estamos utilizando eslint-plugin-React y paquetes de babel-eslint NGP a pelusa ES6 y React código. También hemos instalado a nivel mundial el paquete ESLint NPM."

"We are using eslint-plugin-react and babel-eslint npm packages to lint ES6 and React code. We have also globally installed the ESLint npm package."

"Sí, Shawn. Es común pasar por alto algunas cosas aquí y allí, en un gran proyecto. Tener un comando para pelusa del proyecto nos ayuda en la búsqueda de este tipo de cosas. Necesitamos la eslint, eslint-Babel, y eslint-plugin-React paquetes para utilizar ESLint dentro de nuestro código. Por lo tanto, tenemos que instalarlo antes de ejecutar este comando ".

"Yes, Shawn. It is common to miss a few things here and there in a big project. Having a command to lint the project helps us in finding such things. We need the eslint, eslint-babel, and eslint-plugin-react packages to use ESLint within our code. Therefore, we need to install it before trying to run this command."

npm install eslint --save
npm install babel-eslint --save
npm install eslint-plugin-react –save

"También estamos utilizando algunas opciones de configuración estándar para ESLint. Estas opciones están presentes en el .eslintrc archivo de nuestro proyecto. Definimos las reglas que queremos comprobar con ESLint en este archivo. También activar la función ES6 que queremos ESLint crear una lista blanca. de lo contrario, se elevará errores pelusa de dicho código, ya que sólo es compatible de forma nativa ES5. también especificamos que ESLint debe utilizar babel-eslint como analizador de manera que el código ES6 será analizado por ESLint correctamente."

"We are also using some standard configuration options for ESLint. These options are present in the .eslintrc file of our project. We define the rules that we want to check with ESLint in this file. We also enable ES6 feature that we want ESLint to whitelist. Otherwise, it will raise linting errors for such code as it natively only supports ES5. We also specify that ESLint should use babel-eslint as parser so that ES6 code will be parsed by ESLint properly."
