Chapter 2. JSX in Depth

En el primer capítulo, hemos construido nuestro primer componente utilizando React. Vimos cómo el uso de JSX hace que el desarrollo seá fácil. En este capítulo, vamos a profundizar en JSX.

In the first chapter, we built our first component using React. We saw how using JSX makes the development easy. In this chapter, we will dive deep into JSX.

JavaScript XML ( JSX ) es una sintaxis XML que construye el markup en componentes React. Se puede trabajar en React sin JSX, pero utilizando JSX se hace más fácil de leer y escribir los componentes, así como la estructura de React haciendolo como cualquier otro elemento HTML.

JavaScript XML (JSX) is an XML syntax that constructs the markup in React components. React works without JSX, but using JSX makes it easy to read and write the React components as well as structure them just like any other HTML element.

Al final del capítulo, vamos a familiarizarnos con la sintaxis JSX, la forma en que se debe utilizar con React, y las mejores prácticas de su uso. También vamos a estudiar algunos de los casos de corners que podes ejecutar durante el uso de JSX.

At the end of the chapter, we will get familiar with the JSX syntax, how it should be used with React, and best practices of using it. We will also study some of the corner cases that one can run into while using JSX.

Why JSX?

Shawn tuvo un gran primer día y se le acaba de empezar con el siguiente Adequate Consulting. Con una taza de café, sorprendió a Mike.

Shawn had a great first day and he was just getting started with the next one at Adequate Consulting. With a mug of coffee, he startled Mike.

"Hola Mike, vi que se utilizó para la construcción de nuestra JSX un primer componente. ¿Por qué debemos utilizar JSX cuando React tiene React.createElement?"

"Hey Mike, I saw that we used JSX for building our first component. Why should we use JSX when React has React.createElement?"

"Podes utilizar React sin utilizar JSX. Pero JSX hace que sea fácil de construir componentes React. Se reduce la cantidad de código necesario para escribir. Parece que el formato HTML. Su sintaxis es simple y concisa y son muy fáciles de visualizar los componentes que son construidos cada vez".

"You can use React without using JSX. But JSX makes it easy to build React components. It reduces the amount of code required to write. It looks like HTML markup. Its syntax is simple and concise and it's very easy to visualize the components that are getting built."

"Tomemos un ejemplo de la función de procesamiento de un componente sin utilizar JSX."

"Take an example of the render function of a component without using JSX."

// render without JSX


render: function(){


    return(React.createElement("div", 


                               null, 


                               "Hello React World!"));


}

"Con JSX, se ve mucho mejor."

"With JSX, it looks much better."

// render with JSX


render: function(){


    return <div>


      Hello React World


    </div>;


  }

"En comparación con el ejemplo anterior que no es JSX, el código JSX es mucho más fácil de leer, fácil de entender, y cerca de la marca HTML real."

"Compared to the previous non-JSX example, the JSX code is much more readable, easy to understand, and close to the actual HTML markup."

"La similitud entre JSX y el formato HTML significa que no-desarrolladores como diseñadores de interfaz de usuario y UX en el equipo pueden contribuir en un proyecto utilizando JSX. Tener sintaxis de las etiquetas como-XML también hacen fácil la lectura de grandes árboles de componentes en comparación con las llamadas de función o de objetos literales en JavaScript". explica Mike.

"The similarity between JSX and HTML markup means that non-developers such as UI and UX designers in the team can contribute to a project using JSX. Having XML-like tag syntax also makes reading large component trees easier compared to the function calls or object literals in JavaScript." explained Mike.

"Sí. La sintaxis tiene un aspecto familiar. Nosotros vamos a usar JSX todo el camino en nuestros proyectos?"

"Yeah. The syntax does look familiar. We will be using JSX all the way in our projects right?"

"Sí, lo haremos", comentó Mike.

"Yes, we will", remarked Mike.
