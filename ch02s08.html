Spread attributes

Shawn aprendió un montón de cosas sobre JSX pero cuando él reflexionaba sobre los pasos anteriores, se le ocurrió otra pregunta.

Shawn learned a lot of things about JSX but when he was reflecting on the previous steps, he came up with another question.

"Mike, a partir de ahora sólo estamos pasando dos props del componente App: headings y changesets. Sin embargo, mañana estos Props pueden aumentar a cualquier número y pasarlos uno por uno sería engorroso, especialmente cuando tenemos que pasar algunos datos de los reciente.. cambios en el API directamente. Será difícil mantener una pista de la estructura de los datos entrantes y pasarla en consecuencia en los Props. ¿hay una mejor manera?"

"Mike, as of now we are just passing two props to the App component: headings and changesets. However, tomorrow these props can increase to any number. Passing them one by one would be cumbersome. Especially, when we have to pass some data from the recent changes API directly. It will be hard to a keep track of the structure of the incoming data and pass it accordingly in the props. Is there a better way?"

"Otra excelente pregunta, Shawn. Es cierto que podría ser engorroso pasar una gran cantidad de atributos para el componente de uno en uno. Pero podemos solucionar esto usando los atributos spread".

"Another excellent question, Shawn. True, it might be cumbersome passing a large number of attributes to the component one by one. But we can solve this using the spread attributes."

var props = { headings: headings, changeSets: data, timestamps: timestamps };


ReactDOM.render(<App {...props } />, 


                     document.getElementById('container'));

"En este caso, todas las propiedades del objeto se pasan como Props al componente App . Podemos pasar cualquier objeto que puede contener cualquier número de pares de valores clave y todos ellos se pasarán como Props para el componente" explica Mike.

"In this case, all the properties of object are passed as props to the App component. We can pass any object that can contain any number of key value pairs and all of them will be passed as props to the component" explained Mike.

"Muy bien. El operador (…) está presente sólo en JSX?"

"Very cool. Is the (…) operator present only in JSX?"

"Nop. Se basa en realidad en el elemento de atributo extendido en ES2015, que es el próximo estándar de JavaScript. ES2015 o ES6 que es como se le llama, introduce algunas nuevas características en el lenguaje JavaScript y React se está aprovechando de estas normas en desarrollo con el fin de proporcionar una sintaxis más limpia en JSX" añadió Mike.

"Nope. It is actually based on the spread attribute feature in ES2015, which is the next JavaScript standard. ES2015, or ES6 as it is called, introduces some new features in JavaScript language and React is taking advantage of these developing standards in order to provide a cleaner syntax in JSX" added Mike.

Note

ES2015 ya soporta un operador spread (de difusión) para las arrays en https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_operator. Hay también una propuesta para objetos en https://github.com/sebmarkbage/ecmascript-rest-spread .

ES2015 already supports spread operator for arrays at https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/Spread_operator. There is also a proposal for objects at https://github.com/sebmarkbage/ecmascript-rest-spread.

"No solo esto, los atributos de propagación se pueden usar varias veces o bien se pueden combinar con otros atributos, aunque el orden de los atributos es importante."

"Not only this, spread attributes can be used multiple times or it can be combined with other attributes. The order of the attributes is important though. Newer attributes override previous ones."

var data = [{ "when": "2 minutes ago",


              "who": "Jill Dupre",


              "description": "Created new account"


            },


            {


              "when": "1 hour ago",


              "who": "Lose White",


              "description": "Added fist chapter"


            }];


var headings = ['When', 'Who', 'Description'];





var props = { headings: headings, changeSets: data };





ReactDOM.render(<App {...props} headings = {['Updated at ', 'Author', 'Change']} />, 

document.getElementById('container'));

"En lugar de mostrar Cuando, Quién y Descripción, Actualizado a , Autor, y el cambio se mostrará como los encabezados en este caso", explicó Mike.

"Instead of showing When, Who, and Description, Updated at, Author, and Change will be shown as the headings in this case" Mike explained.

Note

ES2015 o ES6 es el último estándar de la versión JavaScript. Tiene un montón de características, que son utilizadas por React, similares a los atributos de propagación. Nosotros vamos a usar mucho más ES2015 o código ES6 en los próximos capítulos.

ES2015 or ES6 is the latest version JavaScript standard. It has a lot of features, which are used by React, similar to the spread attributes. We will be using lot more ES2015 or ES6 code in the upcoming chapters.
