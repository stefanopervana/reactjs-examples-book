Avoiding state

"En la actualidad, nosotros no necesitamos estado, sin embargo, cuando se traen los datos dinámicos de la API RecentChanges, utilizaremos el estado con los props". Mike añadió.

"Currently, we don't need state; however, when we fetch the dynamic data from RecentChanges API, we will use state with props." Mike added.

"Cool. Sobre la base de nuestra discusión, creo que debemos evitar el estado tanto como sea posible." Shawn sugirió.

"Cool. Based on our discussion, I think that we should avoid state as much as possible." Shawn suggested.

"Es cierto. Si un componente no cambia, entonces no hay necesidad de usar el estado. Es mejor depender de los props pasados por el componente padre en ese caso. Esto también evita volver a representar el componente una y otra vez a medida que los cambios en el estado inician un re-renderizado del componente. "Mike explicó.

"True. If a component does not change, then there is no need to use state. It's better to depend on props passed by the parent component in that case. This also avoids re-rendering of the component again and again as changes to state initiate a re-render of the component." Mike explained.
