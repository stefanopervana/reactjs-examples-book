"Necesitamos este complemento sólo en el modo de desarrollo. Este es un punto importante a recordar, porque en la producción, no necesitamos la información de depuración, ya que podes hacer que nuestra aplicación lenta." informó Mike.

"We need this add-on only in the development mode. This is an important point to remember because in production, we don't need the debugging information as it may make our app slow." informed Mike.

"Shawn, el PERF add-on se podes utilizar para ver qué cambios React está haciendo con el DOM, dónde está gastando el tiempo al intentar mostrar nuestra aplicación, es que perdiendo algo de tiempo mientras que la renderización, y así sucesivamente. Esta información podes ser utilizada para mejorar el rendimiento de la aplicación ". dijo Mike.

"Shawn, the PERF add-on can be used to see what changes React is doing with the DOM, where is it spending time while rendering our app, is it wasting some time while rendering, and so on. This information can then be used to improve the performance of the app." said Mike.

"Vamos a empezar exponiendo el PERF complemento como un objeto global. Podemos utilizarlo en la consola del navegador, mientras que nuestra aplicación se está ejecutando para ver cómo Reactn los está haciendo cambios como por las interacciones de los usuarios." explica Mike.

"Let's start by exposing the PERF add-on as a global object. We can use it in the browser console while our app is running to see how React is making changes as per the user interactions." explained Mike.

// index.js
import ReactDOM from 'react-dom';
import React from 'react';
import App from './App';
import Perf from 'react-addons-perf';

window.Perf = Perf;

ReactDOM.render(<App />, document.getElementById('rootElement'));

"Hemos importado el PERF add-on en nuestro index.js archivo, que es el punto de partida de la aplicación. Podemos acceder al Perf objeto en la consola del navegador como hemos unido a window.Perf". Mike añadió.

"We have imported the PERF add-on in our index.js file, which is the starting point of the app. We can access the Perf object in the browser console as we have attached it to window.Perf." Mike added.

"El PERF complemento viene con métodos que podes ayudar a entender lo que React está haciendo con el DOM cuando cambia algo. Vamos a medir algunas estadísticas de rendimiento. Vamos a empezar el proceso de medición llamando Perf.start() en la consola del navegador. Después de eso, vamos a interactuar con la aplicación. vamos a escribir una consulta para buscar un libro, pulse enviar, y se mostrarán los resultados de búsqueda. vamos a detener la medición del desempeño llamando Perf.stop() en la consola del navegador. Después de eso, vamos a analizar la información que hemos reunido. " Mike explicó el proceso.

"The PERF add-on comes with methods that can help us understand what React is doing with the DOM when something changes. Let's measure some performance statistics. We will start the measurement process by calling Perf.start() in the browser console. After that, we will interact with the app. We will type in a query to search a book, hit submit, and the search results will be displayed. We will stop the performance measurement by calling Perf.stop() in the browser console. After that, let's analyze the information that we gathered." Mike explained the process.
